# Sanitized report examples

These files are deterministic illustrations, not current scan results. All names, packages, paths, rule identifiers, and fingerprints are fictional. They contain no credentials, proprietary source, exploit code, or raw scanner output.

- `minimal-clean.normalized.json`, `minimal-findings.normalized.json`, and `minimal-tool-error.normalized.json` show distinct result categories.
- `standard-mixed-coverage.json`, `standard-not-applicable.coverage.json`, and `standard-not-configured.coverage.json` show coverage states without implying security.
- `standard-tool-error.normalized.json` keeps scanner failure separate from findings.
- `finding-groups.json` and `prioritized-findings.json` show empty valid additive finding-intelligence artifacts.
- `coverage.json`, `normalized.json`, and `report.md` form a readable sample set.
- `sbom.cyclonedx.json` and `sbom.spdx.json` are small valid fictional SBOMs.

Automated tests validate every JSON document and both SBOM structures. Do not upload or baseline these illustrative fingerprints.
